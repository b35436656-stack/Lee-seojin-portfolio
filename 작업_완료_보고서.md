# 작업 완료 보고서

## 변경된 구조

GitHub Pages 정적 사이트를 Google Sheets/Drive 기반 CMS 구조로 개편했습니다.

운영자는 GitHub, HTML, CSS, JavaScript를 몰라도 됩니다.

- Google Sheets: 텍스트, 경력, 포트폴리오, 디자인 설정 관리
- Google Drive: 이미지와 PDF 관리
- GitHub Pages: 사이트 디자인과 코드 배포

## Google Sheets 구조

3개 탭을 사용합니다.

- `Settings`: 기본 정보, 연락처, SNS, 사진/PDF 링크, 디자인 설정, 테마
- `Experience`: 경력과 활동내역
- `Portfolio`: 프로젝트와 포트폴리오 사례

## Google Drive 구조

Drive에는 사진과 PDF를 올리고 공유 링크만 Sheets에 넣습니다.

- 프로필 사진: `Settings.profileImageUrl`
- 배경 이미지: `Settings.backgroundImageUrl`
- 이력서 PDF: `Settings.resumePdfUrl`
- 포트폴리오 이미지: `Portfolio.imageUrl`

## 관리자 운영 흐름

1. Google Sheets에서 문구나 항목을 수정합니다.
2. 사진/PDF는 Google Drive에 올리고 공유 링크를 복사합니다.
3. 복사한 링크를 Google Sheets에 붙여넣습니다.
4. 사이트를 새로고침해 확인합니다.

## 유지보수 방법

- 탭 이름과 열 이름을 바꾸지 않습니다.
- 숨기고 싶은 항목은 삭제하지 말고 `visible`을 `FALSE`로 둡니다.
- 색상/글자 크기 값이 이상하면 값을 비워 기본값으로 복구합니다.
- 민감한 개인정보, 계약서 원본, 자동화 코드 전문은 올리지 않습니다.
